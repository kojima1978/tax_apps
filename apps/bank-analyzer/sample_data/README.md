# サンプルデータについて

このフォルダには、システムの動作確認用のサンプルCSVファイルが含まれています。

## ファイル一覧

1. **三菱UFJ銀行_sample.csv**
   - 三菱UFJ銀行の通帳データサンプル
   - 2024年1月〜5月の取引データ
   - 多額取引（150万円の振込、200万円の入金）を含む

2. **みずほ銀行_sample.csv**
   - みずほ銀行の通帳データサンプル
   - 2024年1月〜5月の取引データ
   - 資金移動の可能性がある取引（2/7に150万円入金）を含む

## CSVフォーマット

両方のサンプルCSVファイルには以下のカラムが含まれています：

```csv
銀行名,支店名,口座番号,年月日,摘要,払戻,お預り,差引残高
```

- **銀行名**: 三菱UFJ銀行 / みずほ銀行
- **支店名**: 青山支店 / 渋谷支店
- **口座番号**: 1234567 / 9876543

これらの情報は自動的に読み取られ、インポート画面で自動入力されます。

## 使い方

1. アプリケーションを起動: `streamlit run main.py`
2. 「案件一覧」から新規案件を作成（例：テスト案件）
3. 「CSVインポート」で各CSVファイルをアップロード
   - **三菱UFJ銀行_sample.csv**
     - CSVから自動読み取り: 銀行名（三菱UFJ銀行）、支店名（青山支店）、口座番号（1234567）
     - 手入力: 名義人（例：山田太郎）
   - **みずほ銀行_sample.csv**
     - CSVから自動読み取り: 銀行名（みずほ銀行）、支店名（渋谷支店）、口座番号（9876543）
     - 手入力: 名義人（例：山田花子）
4. 「分析・表示」で結果を確認

## 注目ポイント

- **資金移動検知**: 三菱UFJ銀行から2/5に150万円出金、みずほ銀行に2/7に150万円入金（日付差2日）
- **多額取引**: 閾値100万円以上の取引が複数存在
- **残高整合性**: 全データで残高が正しく計算されているか確認
