# Bank Analyzer 機能仕様書

## 1. 概要

相続税申告における通帳分析を支援するWebアプリケーション。
複数の銀行口座のCSV/Excelデータを取り込み、資金移動の検出・取引の自動分類・多額取引のハイライトなどを行う。

## 2. 画面仕様

### 2.1 案件一覧（`/`）

| 項目 | 仕様 |
|------|------|
| 機能 | 案件の一覧表示・作成・編集・削除 |
| 検索 | 案件名によるリアルタイムフィルタリング |
| ソート | 新しい順 / 古い順 / 名前順 / 未分類多い順 |
| カード情報 | 案件名、作成日、更新日、取引数、口座数、未分類数 |
| アクション | CSVを取り込む、分析を開く、編集、削除 |
| JSONから復元 | JSONバックアップファイルからの案件復元 |

### 2.2 案件作成・編集（`/new/`, `/case/<id>/edit/`）

| 項目 | 仕様 |
|------|------|
| 入力フィールド | 案件名（必須、ユニーク制約） |
| バリデーション | 空欄不可、重複名不可 |

### 2.3 直接入力（`/case/<id>/direct-input/`）

| 項目 | 仕様 |
|------|------|
| 口座情報 | 銀行名・支店名・種別・口座番号をヘッダーで指定 |
| 既存口座選択 | ドロップダウンで既存口座を選択可能（フィールド自動入力） |
| 入力行 | 5行一括入力（動的追加可能） |
| 行フィールド | 日付（和暦表示対応）、摘要、払戻額、お預り額、残高 |
| 行操作 | チェックボックスで選択→一括削除、行挿入 |

### 2.4 CSVインポートウィザード（`/case/<id>/import/wizard/`）

| 項目 | 仕様 |
|------|------|
| ステップ1 | ファイル選択（CSV/Excel対応） |
| ステップ2 | カラムマッピング・プレビュー |
| ステップ3 | 口座情報入力・確認 |
| エンコーディング | UTF-8, CP932, Shift_JIS 自動検出 |
| 和暦変換 | H28.6.3, R3.4.1 等を西暦に自動変換 |
| 重複検出 | 日付・摘要・金額の組み合わせで重複を警告 |
| 残高整合性 | インポート前に残高の整合性をチェック |

### 2.5 分析ダッシュボード（`/case/<id>/analysis/`）

#### 2.5.1 登録口座一覧
- 口座ごとのカード表示（銀行名、支店名、口座番号、取引件数、最終取引日）
- 口座単位での一括削除

#### 2.5.2 サマリー
- 月次入出金推移チャート（Plotly）
- 出金・入金の棒グラフ

#### 2.5.3 タブ構成

| タブ | 機能概要 |
|------|---------|
| 取引一覧・検索 | 全取引の表示、出金/入金フィルター、金額フィルター（30万/50万/100万以上）、銀行→口座連動フィルター、キーワード検索、分類編集（SaveQueue自動保存）、カテゴリーフィルター絞り込み中に分類変更すると行フェードアウト、複数選択一括変更、付箋 |
| 未分類 | グループ表示（摘要でグルーピング、サジェスト付き）/ フラット表示（個別編集・付箋・一括変更・パターン追加、分類変更後に行自動消去） |
| AI分類 | RapidFuzzファジーマッチングによる分類提案、信頼度スコア表示、個別/一括適用 |
| 資金移動フロー | 検出ペア数・総振替額・未分類ペア数のサマリー、出金元・入金先のペア表示（カード/テーブルビュー切替）、差額（振込手数料）の表示、分類編集、一括「振替」分類 |
| データクレンジング | 重複データ検出・削除、ID範囲指定削除、フィールド値一括置換 |
| 付箋 | 要確認フラグ付き取引の管理、メモ編集（多額取引・取引一覧・未分類タブから付箋可） |
| パターン管理 | 分類キーワードの追加・編集・削除・移動（グローバル/案件固有） |

### 2.6 設定（`/settings/`）

| 設定項目 | 説明 | デフォルト値 |
|---------|------|-------------|
| 多額取引閾値 | この金額以上の取引を多額取引としてハイライト | 500,000円 |
| 資金移動検出期間 | 出金〜入金の日数許容範囲 | 3日 |
| 資金移動許容誤差 | 金額の許容差（振込手数料等） | 1,000円 |
| 日付マッチングモード | `after_only`（出金日以降のみ）/ `both`（前後どちらも） | after_only |
| 贈与判定閾値 | この金額以上の振込を贈与候補として分類 | 1,000,000円 |
| ファジーマッチング有効 | RapidFuzzによる類似度マッチングの有効/無効 | 有効 |
| ファジーマッチング閾値 | 類似度のしきい値（0-100） | 80 |

## 3. 資金移動検出仕様

### 3.1 検出アルゴリズム

1. 全取引を日付順にソート
2. 出金レコードごとに、以下の条件で入金候補を検索:
   - **異なる口座間**の取引
   - **金額が許容誤差内**で一致
   - **日付が指定日数以内**
   - **未マッチ**の入金のみ
3. 複数候補がある場合、**金額差最小→日付差最小**の優先順位で最適候補を選択
4. マッチした出金・入金の両方に`is_transfer=True`をセット

### 3.2 振込手数料の推定
- 出金額 > 入金額の場合、差額を振込手数料として算出
- `transfer_to`フィールドに「手数料XXX円」を付記
- UIでは「差額: XXX円」として表示

### 3.3 日付マッチングモード

| モード | 条件 | 用途 |
|--------|------|------|
| `after_only` | 出金日 ≦ 入金日 ≦ 出金日+N日 | 振込の通常ケース（翌営業日以降に着金） |
| `both` | \|出金日 - 入金日\| ≦ N日 | 同日処理や前日着金のケースもカバー |

## 4. 分類仕様

### 4.1 分類カテゴリー

| カテゴリー | 説明 | 自動判定例 |
|-----------|------|-----------|
| 生活費 | 日常の生活費 | スーパー、光熱費、通信費 |
| 給与 | 給与・賞与・報酬 | 給与振込、賞与 |
| 年金 | 年金関連 | 国民年金、厚生年金 |
| 贈与 | 100万円以上の振込等 | 高額振込（閾値設定可能） |
| 事業・不動産 | 事業関連・不動産収入 | 家賃収入、事業売上 |
| 関連会社 | 関連会社との取引 | 関連法人名の一致 |
| 銀行 | 定期預金・積立等 | 定期、積立、利息 |
| 証券・株式 | 証券会社、配当等 | 証券会社名、配当金 |
| 保険会社 | 生命保険・損保等 | 保険会社名 |
| 通帳間移動 | 口座間の資金移動 | 資金移動検出で自動ペアリング |
| その他 | 手数料、利息、ATM等 | 振込手数料、ATM |
| 未分類 | 未分類の取引 | デフォルト |

### 4.2 分類方法

| 方法 | 説明 | 優先度 |
|------|------|--------|
| ルールベース | キーワードパターンの完全一致 | 高（スコア100） |
| ファジーマッチング | RapidFuzzによる類似度マッチング | 中（スコア90+） |
| 手動分類 | ユーザーによる直接編集 | 最高 |

### 4.3 パターン管理
- **グローバルパターン**: 全案件共通の分類キーワード（`data/user_settings.json`に保存）
- **案件固有パターン**: 案件ごとのカスタムキーワード（`Case.custom_patterns` JSONフィールドに保存）
- パターン操作: 追加・編集・削除・カテゴリ間移動

## 5. インポート・エクスポート仕様

### 5.1 CSVインポート

対応フォーマット:
```
銀行名,支店名,口座番号,年月日,摘要,払戻,お預り,差引残高
```
または最小限:
```
年月日,摘要,払戻,お預り,差引残高
```

対応エンコーディング: UTF-8, CP932, Shift_JIS（自動検出）

### 5.2 エクスポート

| 形式 | 説明 |
|------|------|
| CSV（全データ） | BOM付きUTF-8、Excel互換 |
| CSV（フィルタ済み） | 現在の表示条件でフィルタ |
| Excel（分類別） | 分類ごとにシート分け、和暦変換済み |
| JSON | 案件データの完全バックアップ（設定含む） |

## 6. AJAX API仕様

| エンドポイント | メソッド | 説明 | リクエスト | レスポンス |
|---------------|---------|------|-----------|-----------|
| `/case/<id>/api/toggle-flag/` | POST | 付箋ON/OFF | `{transaction_id}` | `{success, is_flagged}` |
| `/case/<id>/api/create-transaction/` | POST | 取引追加 | フォームデータ | `{success, transaction}` |
| `/case/<id>/api/delete-transaction/` | POST | 取引削除 | `{transaction_id}` | `{success}` |
| `/case/<id>/api/field-values/` | GET | フィールド値一覧 | `?field=<name>` | `{success, values[]}` |

## 7. 技術仕様

### 7.1 レイヤーアーキテクチャ

```
Views → Handlers → Services → Lib → Models → PostgreSQL
```

| レイヤー | 責務 |
|---------|------|
| Views | 画面レンダリング、GETリクエスト処理、テンプレート変数の構築 |
| Handlers | POSTリクエスト処理、AJAX API、バリデーション |
| Services | ビジネスロジック、データ変換、複数モデルにまたがる処理 |
| Lib | 分析エンジン、分類器、ファイルインポーター、設定管理 |
| Models | Django ORM モデル定義（Case, Transaction） |

### 7.2 主要サービス

| サービス | 責務 |
|---------|------|
| `TransactionService` | 取引CRUD、CSV/Excelインポート、一括操作、JSON入出力 |
| `AnalysisService` | 分析データ生成、フィルタリング、ページネーション、統計集計 |

### 7.3 分析エンジン

| モジュール | 責務 |
|-----------|------|
| `analyzer.py` | 多額取引検出、資金移動ペアリング（最適マッチング＋手数料推定） |
| `llm_classifier.py` | RapidFuzzファジーマッチング、ルールベース分類 |
| `importer.py` | CSV/Excel読み込み、エンコーディング検出、和暦変換 |
| `text_utils.py` | NFKC正規化、キーワード検索フィルタリング |

### 7.4 フロントエンド

| ファイル | 責務 |

| `utils.js` | 共通ユーティリティ（postJson, disableButton等） |

| `pattern_manager.js` | パターン管理UIの操作 |
