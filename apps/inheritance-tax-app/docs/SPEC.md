# 相続税シミュレーター 仕様書

## 1. 概要

相続財産額に応じた相続税額をシミュレーションする業務用Webアプリケーション。税理士法人の担当者が顧客への説明・提案に使用することを想定。全UIは日本語。

### 対象ユーザー

- 税理士法人の担当者（PC利用が主、タブレットでの顧客説明も想定）

### 動作環境

- モダンブラウザ（Chrome, Edge, Firefox, Safari）
- Docker コンテナ上で動作（ポート: 3004, basePath: `/inheritance-tax-app`）

## 2. 画面仕様

### 2.1 共通レイアウト

| ポータルリンク | Home アイコン → `/`（ポータルトップ） |
| 会社情報 | 税理士法人マスエージェント（住所・電話番号）※ デスクトップのみ |
| 担当者入力 | 担当者名 + TEL（localStorage キー: `inheritance-tax-staff-info`）※ デスクトップのみ |

- アクティブタブ: 白背景 + 緑テキスト + shadow
- 非アクティブ: 半透明、hover で背景変化

- デスクトップ: 2カラムグリッド（左: 相続人設定、右: 金額入力+オプション）

### 2.2 相続税計算ページ（`/`）

#### 入力

| 項目 | 型 | 単位 | バリデーション |
|------|-----|------|-------------|
| 相続人の構成 | HeirComposition | - | 配偶者または順位いずれか必須 |
| 遺産総額 | number | 万円 | > 0 必須 |
| 配偶者取得割合 | SpouseAcquisitionMode | - | 3モード選択 |

#### 配偶者取得割合モード

| モード | 説明 |
|--------|------|
| `legal` | 法定相続分で取得 |
| `limit160m` | 1億6千万円まで取得 |
| `custom` | 任意額（万円）またはパーセント指定 |

#### 出力（3ページ構成）

**ページ1: 計算結果 + 計算過程**
- サマリーカード4枚: 遺産総額 / 基礎控除 / 課税遺産総額 / 相続税の総額
- ハイライトカード3枚: 納付税額合計 / 相続税負担率 / 加重平均適用税率
- 計算過程ステップ表示

**ページ2: 累進税額の内訳**
- 相続人ごとの超過累進税額の計算内訳

**ページ3: 相続人別 税額内訳**
- 各相続人の法定相続分・按分税額・2割加算・配偶者控除・納付税額

#### 非課税判定

遺産総額 ≤ 基礎控除額の場合、StatusCard（success）で「相続税はかかりません」を表示。

## 3. 計算仕様

### 3.1 基礎控除

```
基礎控除額 = 3,000万円 + 600万円 × 法定相続人数
```

### 3.2 相続税率テーブル（超過累進税率）

| 法定相続分に応じた取得金額 | 税率 | 控除額 |
|:------------------------:|:----:|:-----:|
| 1,000万円以下 | 10% | — |
| 3,000万円以下 | 15% | 50万円 |
| 5,000万円以下 | 20% | 200万円 |
| 1億円以下 | 30% | 700万円 |
| 2億円以下 | 40% | 1,700万円 |
| 3億円以下 | 45% | 2,700万円 |
| 6億円以下 | 50% | 4,200万円 |
| 6億円超 | 55% | 7,200万円 |

### 3.3 法定相続分

| 相続人構成 | 配偶者 | その他（均等分割） |
|-----------|:------:|:---------:|
| 配偶者 + 子 | 1/2 | 1/2 |
| 配偶者 + 直系尊属 | 2/3 | 1/3 |
| 配偶者 + 兄弟姉妹 | 3/4 | 1/4 |

### 3.4 相続税の計算手順

1. **課税遺産総額** = 遺産総額 − 基礎控除額
2. **各人の法定相続分額** = 課税遺産総額 × 法定相続分割合
3. **各人の速算税額** = 法定相続分額 × 税率 − 控除額（速算表適用）
4. **相続税の総額** = 各人の速算税額の合計
5. **各人の按分税額** = 相続税の総額 × (各人の取得額 / 遺産総額)
6. **2割加算** = 按分税額 × 0.2（第3順位の場合）
7. **配偶者控除** = min(按分税額, 相続税の総額 × min(取得額, max(法定相続分額, 1億6千万円)) / 遺産総額)
8. **納付税額** = 按分税額 + 2割加算 − 配偶者控除

### 3.5 死亡保険金の非課税限度額

```
非課税限度額 = 500万円 × 法定相続人数
```

各相続人の非課税額:
```
非課税限度額 × (その相続人の受取保険金額 / 相続人全体の受取保険金合計)
```

### 3.6 保険金シミュレーションの計算モデル

| シナリオ | 遺産額の調整 | 保険金 |
|---------|:------------|:------|
| 現状（既存のみ） | 遺産額そのまま | 既存契約の保険金のみ |
| 提案（既存＋新規） | 遺産額 − 新規保険料合計 | 全契約の保険金 |

```
課税遺産額 = 調整後遺産額 + (保険金合計 − 非課税額) − 基礎控除
```

### 3.7 贈与税の計算（特例贈与税率）

| 基礎控除後の課税価格 | 税率 | 控除額 |
|:------------------:|:----:|:-----:|
| 200万円以下 | 10% | — |
| 400万円以下 | 15% | 10万円 |
| 600万円以下 | 20% | 30万円 |
| 1,000万円以下 | 30% | 90万円 |
| 1,500万円以下 | 40% | 190万円 |
| 3,000万円以下 | 45% | 265万円 |
| 4,500万円以下 | 50% | 415万円 |
| 4,500万円超 | 55% | 640万円 |

```
贈与税基礎控除: 110万円/年/受贈者
課税価格 = 年間贈与額 − 110万円
贈与税額 = 課税価格 × 税率 − 控除額
```

### 3.8 現金贈与シミュレーションの計算モデル

| シナリオ | 遺産額 | 贈与の扱い |
|---------|:------|:----------|
| 現状（贈与なし） | 遺産額そのまま | なし |
| 提案（贈与あり） | 遺産額 − 総贈与額 | 贈与税を別途計算 |

```
総負担 = 相続税 + 贈与税合計
納税後財産額 = 遺産額 − 総負担
```

## 4. データモデル

### 4.1 相続人構成（HeirComposition）

```typescript
interface HeirComposition {
  hasSpouse: boolean;
  selectedRank: 'none' | 'rank1' | 'rank2' | 'rank3';
  rank1Children: Heir[];       // 第1順位: 子
  rank2Ascendants: Heir[];     // 第2順位: 直系尊属
  rank3Siblings: Heir[];       // 第3順位: 兄弟姉妹
}

interface Heir {
  id: string;
  type: HeirType;              // 'child' | 'grandchild' | 'parent' | 'grandparent' | 'sibling' | 'nephew_niece'
  isDeceased?: boolean;        // 代襲相続用
  representatives?: Heir[];    // 代襲相続人（孫・甥姪）
}
```

### 4.2 配偶者取得割合モード（SpouseAcquisitionMode）

```typescript
type SpouseAcquisitionMode =
  | { mode: 'legal' }
  | { mode: 'limit160m' }
  | { mode: 'custom'; value: number; unit: 'amount' | 'percent' };
```

### 4.3 保険契約（InsuranceContract）

```typescript
interface InsuranceContract {
  id: string;
  category: 'existing' | 'new';
  beneficiaryId: string;       // 受取人ID
  beneficiaryLabel: string;
  benefit: number;             // 受取保険金額（万円）
  premium: number;             // 支払保険料（万円）
}
```

### 4.4 贈与受取人（GiftRecipient）

```typescript
interface GiftRecipient {
  id: string;
  heirId: string;
  heirLabel: string;
  annualAmount: number;        // 年間贈与額（万円）
  years: number;               // 贈与年数
}
```

## 6. 印刷仕様

| ページ | 用紙サイズ | 備考 |
|--------|-----------|------|
| 相続税計算 | A4 横向き | 3ページ構成（結果+過程 / 累進内訳 / 相続人別） |

### 印刷時の処理

- `no-print` クラス: 入力エリア・ボタン等を非表示
- `print-only` / `print-only-block`: PrintHeader 等を表示
- `print-page-break`: セクション間の改ページ
- アニメーション無効化（`result-fade-in`）
- shadow / border-radius / padding の縮小
- overflow: visible 強制（テーブル切れ防止）

## 8. 非機能要件

### パフォーマンス

- 全計算はクライアントサイドで即座に実行（サーバー通信なし）
- コンポーネントの memo 化（HeirSettings 等）
- useMemo / useCallback による再計算抑制

### アクセシビリティ

- Radio ボタン: `role="radiogroup"` + `aria-label` + `aria-describedby`
- アイコン: `aria-hidden="true"`
- フォーカスリング: `focus:ring-2 focus:ring-green-500`

### ブラウザ対応

- number input spinner 非表示（CSS）
- `-webkit-overflow-scrolling: touch`（モバイルスクロール）
- スクロールバー非表示（ナビゲーション）
