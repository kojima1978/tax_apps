# 贈与税申告 必要書類案内システム — 仕様書

## 1. システム概要

### 1.1 目的
税理士法人マスエージェントの職員が、贈与税申告に必要な書類一覧をお客様に案内するためのWebアプリケーション。
書類リストの編集・印刷・Excel出力・JSON保存/復元をブラウザ上で完結させる。

### 1.2 利用者
- 税理士法人マスエージェントの職員（担当者）
- 対象: 贈与税申告の顧客対応業務

### 1.3 動作環境
- モダンブラウザ（Chrome, Edge, Firefox, Safari）
- レスポンシブ対応（PC・タブレット・スマートフォン）
- サーバーサイド不要（静的Webアプリケーション）

## 2. 機能要件

### 2.1 初期データ

アプリ起動時に以下の書類カテゴリが自動生成される。

#### 基本必須書類（1カテゴリ）
| カテゴリ | 書類 |
|---------|------|
| 本人確認書類（共通・必須） | 受贈者の本人確認書類、受贈者の利用者識別番号、贈与者の本人確認書類 |

#### 財産の種類別（4カテゴリ）
| カテゴリ | 書類 |
|---------|------|
| 土地・建物 | 固定資産税納税通知書、賃貸借契約書、登記簿謄本 |
| 現金・預貯金 | 贈与契約書、預貯金の通帳・振込明細書 |
| 上場株式 | 贈与契約書、証券会社発行の残高証明書等 |
| 取引相場のない株式 | 贈与契約書、当該法人の決算書等 |

#### 特例関連（4カテゴリ、紫色表示）
| カテゴリ | 注記 |
|---------|------|
| 贈与税の税率の特例 | 課税価格300万円以下の場合は添付不要 |
| 相続時精算課税制度 | 届出書提出済みの場合は添付不要 |
| 配偶者控除の特例 | — |
| 住宅取得等資金の非課税 | — |

### 2.2 書類管理

#### 2.2.1 提出状態管理
- 各書類にチェックボックスあり
- チェックON → 提出済み（取消線表示 + 色をグレーに）
- チェックOFF → 未提出（通常表示）
- カテゴリ単位の「全済み」ボタンで一括ON/OFF

#### 2.2.2 カテゴリ操作
| 操作 | 説明 |
|------|------|
| 追加 | カテゴリ名を入力して追加。通常/特例を選択可能 |
| 削除 | 確認ダイアログ表示後に削除（含まれる書類も全削除） |
| 名前編集 | インライン編集（Enter確定 / Escape取消） |
| 特例切替 | 通常(緑) ↔ 特例(紫)の切替 |
| 展開/折りたたみ | カテゴリ単位、または全カテゴリ一括 |
| 並び替え | ドラッグ&ドロップ |

#### 2.2.3 書類操作
| 操作 | 説明 |
|------|------|
| 追加 | カテゴリ内に書類名を入力して追加 |
| 削除 | 確認ダイアログ表示後に削除 |
| テキスト編集 | インライン編集 |
| チェック切替 | 提出済み/未提出の切替 |
| 並び替え | カテゴリ内でドラッグ&ドロップ |

#### 2.2.4 中項目操作
| 操作 | 説明 |
|------|------|
| 追加 | 書類に対して中項目テキストを入力して追加 |
| 削除 | 確認なしで即削除 |
| テキスト編集 | インライン編集 |

#### 2.2.5 検索
- ツールバーの検索アイコンで検索バーを表示/非表示
- リアルタイムフィルタリング（入力と同時に絞り込み）
- 検索対象: カテゴリ名、書類名、中項目テキスト
- 検索中はカテゴリ追加・書類追加ボタンを非表示
- 検索結果件数を表示
- 検索結果0件時は空状態コンポーネントを表示（クリアボタン付き）

### 2.3 データ入出力

#### 2.3.1 Excel出力
- **ファイル名**: `贈与税申告_必要書類案内_{顧客名}_{YYYYMMDD}.xlsx`
- **内容**: カテゴリ・書類・中項目・チェック状態・注記
- **スタイル**: ヘッダー背景色、罫線、フォント設定、列幅自動調整
- **ライブラリ**: xlsx-js-style

#### 2.3.2 JSON出力
- **ファイル名**: `贈与税申告_書類リスト_{顧客名}_{YYYYMMDD}.json`
- **フォーマット**:
```json
{
  "version": "1.0",
  "exportedAt": "2026-03-08T12:00:00.000Z",
  "staffName": "担当者名",
  "staffPhone": "090-1234-5678",
  "customerName": "お客様名",
  "deadline": "3月末まで",
  "documentList": [ ... ]
}
```

#### 2.3.3 JSON取込
- ファイル選択 → バリデーション → プレビューダイアログ → 確認後にインポート
- **バリデーション**: documentList配列の存在、カテゴリ構造（id, name, documents）、書類構造（id, text, checked）、中項目構造（id, text）
- **ファイルサイズ上限**: 5MB
- バリデーション失敗時はエラーダイアログを表示

#### 2.3.4 印刷
- `window.print()` によるブラウザ印刷機能を使用
- **レイアウト**: 1段組 / 2段組の切替
- **表示モード**: 全書類表示 / 未提出のみ（提出済みを非表示）
- **用紙**: A4、余白10mm
- **ヘッダー**: タイトル、お客様名、資料収集期限、担当者名、発行日
- **フッター**: 会社情報（住所・電話・FAX）、担当者携帯
- **留意事項**: 「電子申告を行う場合、原本資料はご返却いたします。」

#### 2.3.5 リセット
- 確認ダイアログに影響範囲を表示（チェック状態、追加したカテゴリ・書類、中項目、並び順）
- 確認後に初期データに戻す

### 2.4 顧客・担当者情報

#### 2.4.1 入力フィールド
| フィールド | 型 | プレースホルダー | 保存先 |
|-----------|-----|-----------------|--------|
| お客様名 | text | 例：山田 太郎 様 | sessionStorage |
| 資料収集期限 | text | 例：3月末まで | sessionStorage |
| 担当者名 | text | 例：鈴木 一郎 | localStorage |
| 担当者携帯 | tel | 例：090-1234-5678 | localStorage |

#### 2.4.2 永続化ルール
- **localStorage**（担当者名・携帯）: ブラウザを閉じても保持。同一担当者が複数顧客に対応する想定。
- **sessionStorage**（お客様名・期限）: タブを閉じるとクリア。顧客ごとにリセットされる想定。
- ストレージキーのプレフィックス: `gift_tax_`

## 3. 非機能要件

### 3.1 UI/UX

#### 3.1.1 レイアウト
- 全幅レイアウト（`w-full`）で画面全体を活用
- 左右パディング: モバイル 16px / PC 32px

#### 3.1.2 ダークモード
- ツールバーのアイコンボタンでライト/ダーク切替
- 状態は localStorage に保存（キー: `gift_tax_dark_mode`）
- Tailwind CSS の `dark:` バリアントで全コンポーネント対応

#### 3.1.3 レスポンシブデザイン
- **PC (lg以上)**: ツールバーボタンをインライン表示
- **モバイル (lg未満)**: ハンバーガーメニュー（外部クリックで閉じる）

#### 3.1.4 トースト通知
- 操作結果のフィードバックを画面右下に表示
- 種類: success（緑）/ error（赤）/ info（青）
- 3秒で自動消去、手動閉じも可能
- `aria-live="polite"` でスクリーンリーダー対応

#### 3.1.5 アニメーション
| アニメーション | 対象 | 効果 |
|--------------|------|------|
| fadeIn | コンポーネント出現 | 透明度0→1 + 下から10pxスライド（0.3s） |
| toastIn | トースト通知 | 右からスライドイン（0.3s） |

- `prefers-reduced-motion: reduce` でアニメーション無効化対応

#### 3.1.6 アクセシビリティ
- 全インタラクティブ要素に `aria-label` を付与
- チェックボックスに `role="checkbox"` + `aria-checked`
- カテゴリカードに `role="region"` + `aria-label`
- 展開/折りたたみに `aria-expanded`
- ドラッグハンドルに `aria-roledescription`
- ダイアログに `role="dialog"` / `role="alertdialog"` + `aria-modal`

#### 3.1.7 カラーテーマ
| 要素 | ライト | ダーク |
|------|--------|--------|
| 背景 | #ffffff | #0f172a (slate-900) |
| テキスト | #171717 | #e2e8f0 (slate-200) |
| 通常カテゴリ | emerald-50 / border-emerald-500 | emerald-950 |
| 特例カテゴリ | purple-50 / border-purple-500 | purple-950 |
| アクションボタン | emerald-600 | emerald-600 |
| 提出済み | slate-400 + line-through | slate-500 + line-through |

### 3.2 パフォーマンス

- コンポーネントの `React.memo` によるメモ化
- ハンドラーの `useCallback` による参照安定化
- ハンドラーオブジェクト（docHandlers, subItemHandlers, categoryHandlers）の `useMemo` による一括メモ化
- 検索結果の `useMemo` によるフィルタリング結果キャッシュ
- リスト操作は全てイミュータブルな純粋関数（editableListUtils.ts）

### 3.3 データ永続化

| データ | 保存先 | タイミング |
|--------|--------|-----------|
| ダークモード | localStorage | トグル時 |
| 担当者名 | localStorage | 入力変更時 |
| 担当者携帯 | localStorage | 入力変更時 |
| お客様名 | sessionStorage | 入力変更時 |
| 資料収集期限 | sessionStorage | 入力変更時 |
| 書類リスト | なし（揮発） | JSON出力で手動保存 |

> 書類リストはブラウザストレージに自動保存されない。データ保存はJSON出力で明示的に行う設計。

## 5. 状態管理アーキテクチャ

```
EditableListStep（統合ビュー）
├── useGiftTaxGuide        … アプリ全体（印刷設定、ストレージ永続化、Excel出力）
├── useEditableListEditing  … 編集オーケストレーター
│   ├── useDeleteConfirm       … 削除確認ダイアログ
│   ├── useDocumentEditing     … 書類編集
│   ├── useSubItemEditing      … 中項目編集
│   ├── useCategoryEditing     … カテゴリ編集
│   └── useJsonImportExport    … JSON入出力
├── useDragAndDrop          … ドラッグ&ドロップ
├── useDarkMode             … ダークモード
└── useToast                … トースト通知
```

- 全状態はReactのuseState/useRefで管理（外部ストアなし）
- hookの分割はドメイン責務単位（書類/中項目/カテゴリ/削除/JSON）
- `useEditableListEditing` がサブhookを統合し、返り値をスプレッドで公開
- 消費側（EditableListStep）のインターフェースは変わらない
