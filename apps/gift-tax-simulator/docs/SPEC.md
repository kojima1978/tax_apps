# 贈与税 比較Webアプリ — 仕様書

## 1. 概要

### 1.1 目的
税理士法人マスエージェント向けの業務支援ツール。贈与税・不動産取得税・登録免許税の試算を行い、顧客への説明資料（印刷）として活用する。

### 1.2 対象ユーザー
- 税理士法人の職員
- 顧客への説明時に利用

### 1.3 動作環境
- モダンブラウザ（Chrome, Edge, Firefox, Safari）
- レスポンシブ対応（PC・タブレット・スマートフォン）
- サーバーサイド処理なし（全計算をクライアントサイドで実行）

## 3. 機能仕様

### 3.1 贈与税シミュレーター（`/`）

#### 入力項目

| 項目 | 型 | 必須 | バリデーション | 備考 |
|-----|---|------|-------------|------|
| 贈与金額 | 数値（円） | Yes | 1円〜10億円 | カンマ区切り自動フォーマット、全角→半角変換 |
| 贈与区分 | 選択 | Yes | - | 特例贈与 / 一般贈与 |

#### 出力

| 項目 | 説明 |
|-----|------|
| パターン名 | 一括贈与 / 2年分割 / 4年分割 |
| 1回あたりの贈与額 | 総額 ÷ 分割回数（端数切捨） |
| 1回あたりの税額 | 速算表による計算 |
| 申告回数 | 分割回数 |
| 実効税率 | 1回あたり税額 ÷ 1回あたり贈与額 |
| トータル贈与税額 | 1回あたり税額 × 分割回数 |
| 棒グラフ | トータル贈与税額のパターン間比較 |

#### 空状態
- 未入力・未計算時はアイコンとガイドテキストを表示

#### 操作
- 金額入力 → 区分選択 → 「計算する」ボタン押下
- Enterキーでも計算実行

贈与税シミュレーターと同一（金額・区分）

#### 出力

| 項目 | 説明 |
|-----|------|

#### 行スタイル

#### 空状態

### 3.4 不動産取得税シミュレーター（`/acquisition-tax`）

#### 入力項目

**共通設定**

| 項目 | 型 | 選択肢/備考 |
|-----|---|------------|
| 登記原因（取引種別） | 選択 | 売買 / 新築 / 相続 / 贈与 |
| 計算対象 | トグル | 土地 / 建物（複数選択可） |

**土地の情報**

| 項目 | 型 | 備考 |
|-----|---|------|
| 宅地 - 固定資産税評価額 | 数値（円） | 宅地特例（1/2）適用 |
| 宅地 - 面積（m²） | 数値 | 税額軽減計算に使用 |
| その他（宅地以外） - 評価額 | 数値（円） | 特例なし |

**建物の情報**

| 項目 | 型 | 備考 |
|-----|---|------|
| 固定資産税評価額 | 数値（円） | |
| 床面積（m²） | 数値 | 土地の税額軽減にも使用 |
| 建築年月日 | 年/月/日 選択 | 1900年〜現在 |
| 居住用かどうか | チェック | 税率・控除に影響 |
| 控除額 | 数値（円） | 建築年月日から自動計算 |

#### 計算ロジック

**土地（宅地）**
1. 課税標準額 = 評価額 × 1/2（宅地特例）
2. 税額 = 課税標準額 × 3%
3. 住宅用地の税額軽減:
   - 控除額A = 45,000円
   - 控除額B = (課税標準額 ÷ 面積) × min(床面積×2, 200) × 3%
   - 適用控除 = max(A, B)
4. 納付税額 = 税額 - 適用控除（0円以上）

**土地（その他）**
1. 課税標準額 = 評価額（特例なし）
2. 税額 = 課税標準額 × 3%

**建物**
1. 課税標準額 = 評価額 - 控除額
2. 税率 = 3%（住宅用）/ 4%（非住宅）
3. 税額 = 課税標準額 × 税率

**建築年月日による控除額**

| 建築時期 | 控除額 |
|---------|--------|
| 1997年4月1日以降 | 1,200万円 |
| 1989年4月1日〜 | 1,000万円 |
| 1985年7月1日〜 | 450万円 |
| 1981年7月1日〜 | 420万円 |
| 1976年1月1日〜 | 350万円 |
| 新築 | 1,200万円（原則） |

**相続の場合**: 不動産取得税は非課税

#### 出力
- 結果ボックス: 土地（宅地/その他）・建物の内訳 + 合計
- 計算過程の詳細（トグル展開）

---

### 3.5 登録免許税シミュレーター（`/registration-tax`）

#### 入力項目

**共通設定**: 不動産取得税と同一（取引種別・計算対象）

**土地の情報**

| 項目 | 型 |
|-----|---|
| 固定資産税評価額 | 数値（円） |

**建物の情報**

| 項目 | 型 | 備考 |
|-----|---|------|
| 固定資産税評価額 | 数値（円） | |
| 居住用かどうか | チェック | |
| 住宅用家屋証明書あり | チェック | 居住用の場合のみ表示 |

#### 計算ロジック

**土地**
1. 課税標準額 = 評価額（千円未満切捨）
2. 税率:
   - 売買: 1.5%（特例）
   - 相続: 0.4%
   - 贈与: 2.0%
   - 新築: 0.4%
3. 税額 = 課税標準額 × 税率（百円未満切捨、最低1,000円）

**建物**
1. 課税標準額 = 評価額（千円未満切捨）
2. 税率:

| 取引種別 | 通常 | 住宅用家屋証明あり |
|---------|------|-----------------|
| 売買 | 2.0% | 0.3% |
| 新築（保存登記） | 0.4% | 0.15% |
| 相続 | 0.4% | - |
| 贈与 | 2.0% | - |

3. 税額 = 課税標準額 × 税率（百円未満切捨、最低1,000円）

#### 出力
- 結果ボックス: 土地・建物の内訳 + 合計
- 計算過程の詳細（トグル展開）

## 4. 共通機能

### 4.1 ナビゲーション（inheritance-tax-appと同パターン）
- フラットデザイン（`border-b` + `backdrop-blur`）、1行 h-14 コンパクト
- 左: ポータルリンク（Home + 「ポータル」）
- 中央: pill型ナビリンク5つ（active: `bg-emerald-50 text-emerald-700`）
- 右: 担当者入力（lg以上）+ 印刷ボタン
- モバイル: アイコン+短縮ラベル表示

### 4.2 評価額引用（ページ間データ連携）
- 不動産取得税 → 登録免許税、またはその逆方向で評価額を引用可能
- localStorage経由でページ間のデータを共有
- 引用ボタンはインポートバーに配置（印刷時非表示）

### 4.3 数値入力
- 全角数字は自動で半角に変換
- カンマ区切りを自動フォーマット
- 数値以外の文字は自動除去
- 金額入力フィールドはRoboto Monoフォント（等幅）

### 4.4 印刷
- 各ページにA4印刷対応スタイル（早見表はA3）
- 印刷時:
  - ナビゲーションタブ・計算ボタン非表示
  - ヘッダーは透明背景 + 緑ボーダーに変更
  - ページタイトル（税目名）をヘッダーに表示
  - フッターに会社情報・担当者名・作成日を表示
  - 不動産ページは計算過程を自動展開
- 印刷ボタンはナビゲーションバー右端に配置

### 4.5 会社情報
- 税理士法人マスエージェント
- 印刷用フッターに表示

### 4.6 担当者情報
- ヘッダー右側に入力欄（lg以上で表示）
- localStorage に保存（キー: `gift-tax-staff-info`）
- 印刷時フッターに反映

## 5. UI/UXデザイン

### 5.1 テーマ
- プライマリカラー: 緑系（emerald）
- ヘッダー: フラットデザイン（border-b + backdrop-blur、inheritance-tax-app準拠）
- 特例贈与: 青系（`#1565c0`）
- 一般贈与: オレンジ系（`#e65100`）

### 5.2 フォント
- 本文: Noto Sans JP（400, 500, 700）
- 数値: Roboto Mono

### 5.3 レイアウト
- 贈与税ページ: 入力（上）→ 結果テーブル + グラフ（下）
- 不動産税ページ: 共通設定（上）→ 土地/建物 2カラム入力（中）→ 計算ボタン（下）→ 結果（下）
- 入力と結果は同一ページ内でスクロール

### 5.4 レスポンシブ
- 768px以下:
  - ヘッダータイトル縮小、サブタイトル非表示
  - 会社情報非表示
  - タブは横スクロール
  - 不動産ページの2カラム→1カラム
  - グラフ高さ200px

### 5.5 空状態
- 贈与税・年数比較ページで未計算時に表示
- アイコン（48px、線幅1.2）+ ガイドテキスト
- グレー色（`#9ca3af`）、中央揃え

## 6. データフロー

### 6.1 状態管理
- React useState / useCallback によるローカル状態管理
- 外部状態管理ライブラリ不使用

### 6.2 永続化
- localStorage: 評価額のページ間引用データのみ
- データベース: なし（全データはクライアントサイドのみ）

### 6.3 バンドル分割
- `react`, `react-dom` → react チャンク
- `react-router-dom` → react-router チャンク
- `chart.js`, `react-chartjs-2` → chartjs チャンク（lazy load）

---

## 7. 非機能要件

### 7.1 パフォーマンス
- Chart.jsコンポーネントはlazy load（React.lazy + Suspense）
- lucide-reactは個別アイコンインポート（バンドルサイズ最適化）
- manualChunksによるバンドル分割

### 7.2 セキュリティ（本番nginx）
- X-Frame-Options: SAMEORIGIN
- X-Content-Type-Options: nosniff
- Referrer-Policy: strict-origin-when-cross-origin
- Permissions-Policy: camera/microphone/geolocation無効
- CSP: self + Google Fonts

### 7.3 入力バリデーション
- 贈与金額: 1円〜10億円
- 評価額: 1円〜100億円
- 計算結果の異常値チェック（isFinite, isNaN）
- 不正入力時はエラーメッセージ表示
