# 相続税申告 資料準備ガイド — 仕様書

## 1. 概要

### 1.1 目的

相続税申告に必要な資料の準備・確認・管理をサポートするWebアプリケーション。税理士事務所のスタッフが、顧客ごとに必要書類の収集状況を一覧管理し、印刷・Excel出力・JSON保存/復元が可能。

### 1.2 ユーザー

税理士事務所のスタッフ（担当者）

### 1.3 動作環境

- モダンブラウザ（Chrome、Edge、Firefox、Safari）
- Docker環境（開発・本番）

## 2. 書類リスト種別

1ページ内でドロップダウン切替。種別ごとにlocalStorageで自動保存。

| 種別キー | ラベル | 説明 |
|----------|--------|------|
| `inheritance-tax` | 相続税申告 | 相続税申告に必要な全書類の管理（11カテゴリ） |
| `simplified` | 相続シミュレーション | シミュレーション用に必要最小限の書類（1カテゴリ） |
| `unlisted-stock` | 非上場株式評価 | 非上場株式評価に必要な書類の管理（4カテゴリ） |

種別切替時、現在のデータはlocalStorageに保存され、切替先のデータが復元される。

- **テーマカラー**: 緑（emerald）ベース — tax-docs と統一
- **背景**: グラデーション（slate → emerald → slate）、ダークモード対応
- **コンテンツ幅**: `max-w-7xl`（80rem）中央配置
- **カード型UI**: カテゴリごとの展開/折畳カード + 書類カード

| 連絡先 | テキスト入力（localStorage自動保存） |

| 要素 | 仕様 |

| 提出済み非表示 | 印刷時に提出済み書類を非表示にするトグル |

### 3.3 カテゴリカード（SortableCategoryCard）

各カテゴリはドラッグ&ドロップ対応のカードとして表示。

#### カテゴリヘッダー

| 要素 | 仕様 |
|------|------|
| ドラッグハンドル | GripVertical アイコン。カテゴリの並び替えに使用 |
| 展開/折畳 | ChevronDown / ChevronRight アイコン。クリックで切替 |
| カテゴリ番号 | 丸数字（①②③...） |
| カテゴリ名 | インライン編集可能（編集ボタンで開始） |
| 無効化ボタン | カテゴリ全体を対象外に設定（グレー表示） |
| 編集ボタン | カテゴリ名のインライン編集を開始 |
| 削除ボタン | カテゴリ削除（確認ダイアログ付き） |
| 全済みボタン | カテゴリ内全書類の提出済みを一括切替 |

#### 進捗バー

- カテゴリヘッダー直下に表示
- チェック済み / 全体（対象外除く）の割合
- 全完了時は色が変わる

### 3.4 書類カード（SortableDocumentItem）

各書類はドラッグ&ドロップ対応のカードとして表示。

#### メイン行

| 要素 | 仕様 |
|------|------|
| ドラッグハンドル | GripVertical アイコン |
| チェックボックス | 提出済みチェック。チェック時に提出日を自動記録 |
| 書類番号 | 太字表示 |
| 書類名 | 提出済みは取消線 + 薄色表示 |
| 説明 | 書類名の下に小さめグレー文字で表示 |
| 入手方法 | さらに小さめで表示（折畳/展開可能） |
| バッジ | 委任可（青）、急（赤）、対象外（グレー）、追加（黄）、提出日（緑） |

#### アクションボタン

| ボタン | 説明 |
|--------|------|
| 委任可切替 | 取得代行可否をトグル |
| 急切替 | 緊急フラグをトグル |
| 対象外切替 | 対象外フラグをトグル |
| 編集 | 書類の名前・説明・取得方法を編集（モーダル） |
| 個別名追加 | 個別名（サブアイテム）を追加 |
| 削除 | 書類削除（確認ダイアログ付き） |

#### 個別名サブアイテム

- 親子連番形式（例: 1-1, 1-2）
- CornerDownRight アイコンで階層を視覚化
- インライン編集・削除可能

### 3.5 書類追加/編集モーダル

| 項目 | 型 | 説明 |
|------|-----|------|
| 書類名 | text | 必須 |
| 内容説明 | textarea | 任意 |
| 取得方法 | textarea | 任意 |

- 追加モード: カテゴリに新規カスタム書類を追加
- 編集モード: 既存書類の名前・説明・取得方法を変更

### 3.6 ダイアログ

| ダイアログ | 用途 |
|-----------|------|
| 削除確認 | 書類・カテゴリの削除前に確認 |
| リセット確認 | 書類カスタマイズを初期状態に戻す前に確認 |
| インポート確認 | JSONファイル読込前に確認（既存データが上書きされる旨） |
| インポートエラー | 無効なJSONファイルの場合に表示 |

### 3.7 Toast通知

操作結果をポップアップで通知（3秒で自動消去）。

| タイプ | 用途 |
|--------|------|
| success | Excel出力成功、JSON保存成功、インポート成功 |
| error | Excel出力失敗 |
| info | リセット完了 |

### 4.4 データモデル

#### ソースデータ（マスターデータ）

```typescript
interface DocumentItem {
  id: string;
  name: string;
  description: string;
  howToGet: string;
  canDelegate?: boolean;
}

interface CategoryData {
  id: string;
  name: string;
  documents: DocumentItem[];
}
```

#### ランタイムデータ（編集可能）

```typescript
interface SpecificName {
  id: string;
  text: string;
}

interface EditableDocument {
  id: string;
  name: string;
  description: string;
  howToGet: string;
  canDelegate: boolean;
  checked: boolean;
  checkedDate?: string;
  excluded: boolean;
  urgent: boolean;
  specificNames: SpecificName[];
  isCustom: boolean;
}

interface EditableCategory {
  id: string;
  name: string;
  documents: EditableDocument[];
  isExpanded: boolean;
  isDisabled: boolean;
}

type EditableDocumentList = EditableCategory[];
```

ソースデータ（`CategoryData[]`）は初期化時に `EditableDocumentList` に変換される。すべての状態はドキュメントオブジェクト内に統合（overlay-maps不使用）。

## 5. 機能仕様

### 5.1 書類管理

| 機能 | 説明 |
|------|------|
| 書類の編集 | モーダルダイアログで書類名・説明・取得方法をカスタマイズ |
| カスタム書類追加 | 独自の書類をカテゴリに追加（「＋書類を追加」ボタン） |
| カテゴリ追加 | 新規カテゴリを追加 |
| 個別名（具体名） | サブアイテムとして具体的な書類名を追加 |
| ドラッグ&ドロップ | カテゴリ・書類それぞれD&Dで並び順を変更 |
| 代行可否設定 | 各書類の取得代行可否をワンクリック切替 |
| 提出済みチェック | 書類ごとのチェック + 提出日自動記録 + カテゴリ単位の全済みボタン |
| 緊急フラグ | 書類に緊急マークを設定（赤バッジ、印刷時も強調） |
| 対象外設定 | 不要な書類を対象外に（半透明+バッジ、印刷・進捗から除外） |
| カテゴリ無効化 | カテゴリ単位で対象外に設定（グレー表示、印刷時は非表示） |
| 書類・カテゴリ削除 | 確認ダイアログ付きの削除 |
| 初期化 | 書類カスタマイズを標準状態に戻す（確認ダイアログ付き） |

### 5.2 状態管理

| 機能 | 説明 |
|------|------|
| localStorage自動保存 | 種別ごとにデータを自動保存・復元 |
| ダークモード | ワンクリック切替、localStorage保存 |
| Toast通知 | 操作結果をポップアップ通知 |

### 5.3 出力機能

#### Excel出力

- xlsx-js-style によるスタイル付きExcelファイル生成
- カテゴリヘッダー行（セル結合）
- テーブルヘッダー: ✓ / 必要書類名 / 内容説明 / 取得方法 / 代行
- 提出済みは `[済]` プレフィックス + ☑ マーク
- 緊急書類は `【急】` プレフィックス + 赤背景セル
- 個別名は連番付きで出力
- ヘッダーに基本情報（お客様名、被相続人名、担当者、連絡先）

#### 印刷（リスト型レイアウト）

- **1行目**: チェック□ + 番号 + 書類名 + バッジ（急/取得代行可/追加）
- **2行目**: 説明（小さめグレー文字）
- **3行目**: 入手方法（「入手:」ラベル付き）
- 急フラグの書類は赤背景で強調表示
- 個別名は親子連番形式（1-1, 1-2）で表示
- 1列/2列レイアウト切替対応
- 提出済み書類の非表示オプション
- ヘッダー: タイトル・依頼者名・被相続人名・担当者・発行日・事務所名
- フッター: 留意事項・事務所住所・連絡先

#### JSON保存/読込

- 設定をJSONファイルとして保存
- JSONファイルから設定を復元
- バージョン管理: v2.0.0（EditableDocumentList形式）
- 旧フォーマット（v1.0 overlay-maps形式）の後方互換あり

## 7. 非機能要件

### 7.1 パフォーマンス

- Viteによる高速ホットリロード（開発時）
- React.memo による不要な再レンダリング防止（SortableCategoryCard, SortableDocumentItem）
- useMemo / useCallback によるコールバック安定化
- manualChunks: react / dndkit バンドル分離
- useRef による最新値参照（DnDハンドラーの依存配列安定化）

### 7.2 データ保全

- localStorage自動保存（種別ごとに独立キー）
- JSON保存時のバージョニング（v2.0.0）
- 旧データ形式の後方互換（v1.0 overlay-maps → v2.0 EditableDocumentList 変換）
- useToast のタイマーリーク防止（アンマウント時クリーンアップ）

### 7.3 印刷対応

- `hidden print:block` によるレイアウト切替
- リスト型レイアウト（1列/2列対応）
- 印刷専用ヘッダー（事務所情報・担当者連絡先）
- 印刷専用フッター（留意事項・事務所住所）
- 操作ボタン等は印刷時に非表示（`no-print` クラス）
- 提出済み書類の印刷非表示トグル
- `break-inside-avoid` でカテゴリの途中改ページを防止

### 7.4 アクセシビリティ

- セマンティックHTML: `role`, `aria-expanded`, `aria-checked`, `aria-label`
- キーボード操作: Enter/Space で展開、Escape でキャンセル
- ドラッグ&ドロップ: キーボードセンサー対応（`sortableKeyboardCoordinates`）
- モーダル: フォーカストラップ、Escape で閉じる
