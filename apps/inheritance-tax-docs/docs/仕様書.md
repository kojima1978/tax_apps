# 相続税申告 資料準備ガイド — 仕様書

## 1. 概要

### 1.1 目的

相続税申告に必要な資料の準備・確認・管理をサポートするWebアプリケーション。税理士事務所のスタッフが、顧客ごとに必要書類の収集状況を一覧管理し、印刷・Excel出力・JSON保存/復元が可能。

### 1.2 ユーザー

税理士事務所のスタッフ（担当者）

### 1.3 動作環境

- モダンブラウザ（Chrome、Edge、Firefox、Safari）
- Docker環境（開発・本番）
- ローカル開発（Node.js 22以上）

## 2. 画面仕様

### 2.1 レイアウト

1画面完結型（ルーティングなし）。統合ビュー方式でテーブル上のWYSIWYG編集。

### 2.2 ヘッダーツールバー

グラデーション背景（青系135deg）のツールバー。

| 要素 | 仕様 |
|------|------|
| Homeボタン | アイコン → `/`（ポータルトップ） |
| タイトル | 「相続税申告 資料準備ガイド」 |
| 保存ボタン | 設定をJSONファイルとして保存（Ctrl+S対応）。未保存時は「未保存」バッジ表示 + 背景色がamberに変化。保存済み時は最終保存時刻を表示 |
| 読込ボタン | JSONファイルから設定を復元 |
| Excelボタン | スタイル付きExcelファイルを生成・ダウンロード |
| 印刷ボタン | ブラウザ印刷ダイアログ |
| 初期化ボタン | 書類カスタマイズを標準状態に戻す（確認ダイアログ付き、基本情報は保持） |

### 2.3 基本情報入力

| 項目 | 型 | 説明 |
|------|-----|------|
| お客様名 | text | 顧客名 |
| 被相続人名 | text | 亡くなった方の名前 |
| 資料収集期限 | date | 期限日 |
| 担当者 | text | 担当者名 |
| 担当者連絡先 | text | 電話番号等 |

### 2.4 ツールバー（フィルター・検索）

| 要素 | 仕様 |
|------|------|
| 全展開 / 全折りたたみ | 全カテゴリの展開・折りたたみ切替 |
| 未提出のみ | 未チェックの書類のみ表示 |
| 代行可のみ | 取得代行可能な書類のみ表示 |
| 緊急のみ | 緊急フラグ付きの書類のみ表示 |
| 対象外を非表示 | 対象外書類を非表示 |
| 書類名検索 | テキスト入力によるリアルタイムフィルタ |
| 提出済み印刷非表示 | 提出済み書類を印刷時に非表示にするトグル |

- アクティブなフィルターがある場合はリセットボタンを表示

### 2.5 注意事項セクション

以下の案内を表示:
- 原本が必要な書類についての注意
- 取得代行可能な書類についての案内
- 身分関係書類についての補足

### 2.6 カテゴリテーブル

各カテゴリごとに展開・折りたたみ可能なテーブルを表示。

#### カテゴリヘッダー

| 要素 | 仕様 |
|------|------|
| アイコン | Lucideアイコン（カテゴリ固有） |
| カテゴリ番号 | 丸数字（①②③...） |
| カテゴリ名 | |
| 緊急バッジ | カテゴリ内に緊急書類がある場合、赤バッジで「急 N」を表示 |
| 全済みボタン | 3段階切替（未チェック → 全チェック → 全解除） |
| 無効化ボタン | カテゴリ全体を対象外に設定 |
| 削除ボタン | カテゴリ削除（確認ダイアログ付き） |
| カラーアクセント | 左ボーダーにカテゴリ固有色 |

#### 書類行（テーブル行）

| 列 | 説明 |
|----|------|
| D&Dハンドル | ドラッグ&ドロップ用グリップ |
| チェックボックス | 提出済みチェック。チェック時に提出日を自動記録 |
| 書類番号 + 書類名 | 番号は太字。バッジ群: カスタム / 編集済み / 代行可 / 対象外 / 緊急 / 提出日 |
| 内容説明 | 書類の説明文 |
| 取得方法 | 書類の取得手順 |
| 代行 | 取得代行可否アイコン（ワンクリック切替） |
| 操作 | メモ / 緊急切替 / 対象外切替 / 編集 / 削除 |

- 提出済み行: 取消線 + テキスト薄表示 + 提出日バッジ
- 対象外行: 半透明 + 「対象外」バッジ
- 緊急行: 左ボーダー赤 + 「急」バッジ（赤）
- 書類名クリックで編集モーダルを開く

#### 具体的書類名サブ行

| 要素 | 仕様 |
|------|------|
| 親子連番 | 親書類番号-枝番形式（例: 1-1, 1-2, 1-3） |
| 表示 | colSpan=3で3列にまたがり表示 |
| 連続入力 | Enter/Tab/Tabキーで次の入力欄に移動、最後でEnterで新規追加 |
| テキストクリック編集 | テキストをクリックしてインライン編集 |
| D&D並べ替え | 具体名行の並び替え |
| 削除 | 個別削除ボタン |

### 2.7 書類追加/編集モーダル

| 項目 | 型 | 説明 |
|------|-----|------|
| 書類名 | text | 必須 |
| 内容説明 | textarea | 任意 |
| 取得方法 | textarea | 任意 |

- 追加モード: カテゴリに新規カスタム書類を追加
- 編集モード: 既存書類の名前・説明・取得方法を変更

### 2.8 確認ダイアログ

- 書類削除、カテゴリ削除、初期化操作で表示
- タイトル + メッセージ + キャンセル/確認ボタン

### 2.9 留意事項・フッター

- 事務所住所・連絡先を表示（税理士法人マスエージェント）
- 印刷時にも表示

## 4. 機能仕様

### 4.1 統合ビュー（WYSIWYG編集）

テーブル上で直接操作可能なWYSIWYG型の画面構成。

| 機能 | 説明 |
|------|------|
| 書類の編集 | モーダルダイアログで書類名・説明・取得方法をカスタマイズ |
| カスタム書類追加 | 独自の書類をカテゴリに追加（「＋追加」ボタン） |
| 具体的書類名 | テーブルサブ行で具体名を表示（連続入力・テキストクリック編集・D&D並べ替え） |
| ドラッグ&ドロップ | 書類行・具体名行それぞれD&Dで並び順を変更（ネストDndContext） |
| 代行可否設定 | 各書類の取得代行可否をワンクリック切替 |
| 提出済みチェック | 書類ごとのチェックボックス + 提出日自動記録 + カテゴリ単位の「全済み」3段階ボタン |
| メモ | 各書類にメモを追加・編集（印刷時にも表示） |
| 緊急フラグ | 書類に緊急マークを設定（赤バッジ表示、Excel出力に【急】プレフィックス） |

| 対象外設定 | 不要な書類を対象外に（半透明+バッジ表示、印刷にも反映） |
| カテゴリ無効化 | カテゴリ単位で対象外に設定（印刷時は非表示） |
| 書類・カテゴリ削除 | 確認ダイアログ付きの削除 |
| フィルター/検索 | 未提出のみ・代行可のみ・緊急のみ・対象外非表示・書類名検索 |
| 初期化 | 書類カスタマイズを標準状態に戻す（確認ダイアログ付き、基本情報は保持） |

### 4.2 未保存状態管理

| 機能 | 説明 |
|------|------|
| 変更検知 | 書類・基本情報の変更を追跡（isDirtyフラグ） |
| 未保存バッジ | 保存ボタンに「未保存」バッジ + amber背景で視覚的に通知 |
| 最終保存時刻 | 保存ボタンに最終保存時刻を表示（例: 「保存 (14:30:25)」） |
| ページ離脱警告 | 未保存変更がある状態でページを閉じる際にbeforeunload警告 |

### 4.3 出力機能

#### Excel出力

- xlsx-js-style によるスタイル付きExcelファイル生成
- カテゴリヘッダー行（背景色付き）
- 書類行: 番号 + 書類名 + 説明 + 取得方法 + チェック状態
- 提出済みは `[済]` プレフィックス + ☑ マーク
- 緊急書類は `【急】` プレフィックス + 赤背景セル
- 具体的書類名は連番付きで出力
- ヘッダーに基本情報（お客様名、被相続人名、期限、担当者、連絡先）
- ファイル名: `相続税_資料準備ガイド_{お客様名}.xlsx`

#### 印刷/PDF保存

- ブラウザ印刷機能（A4横向き）
- 印刷専用ヘッダー: タイトル・発行日・事務所名・基本情報
- 書類番号を太字で強調
- 具体名は親子連番形式（1. → 1-1, 1-2）
- 提出済みチェックマーク表示
- 提出済み書類を印刷非表示にするトグル対応
- カラーアクセント（左ボーダー）は印刷時に非表示
- 操作ボタン・フィルター等は印刷時に非表示

#### JSON保存/読込

- 設定をJSONファイルとして保存（Ctrl+S ショートカット対応）
- JSONファイルから設定を復元（旧データの後方互換あり）
- バージョン管理: v1.0.0

## 6. 非機能要件

### 6.1 パフォーマンス

- Viteによる高速ホットリロード（開発時）
- useMemo / useCallback による不要な再レンダリング防止
- manualChunks: react / dndkit バンドル分離

### 6.2 データ保全

- beforeunload による未保存変更の警告
- isDirty フラグによる変更追跡
- JSON保存時のバージョニング（v1.0.0）
- 旧データ形式の後方互換（strikethroughDocuments → checkedDocuments マイグレーション）

### 6.3 印刷対応

- `@media print` によるレイアウト最適化（A4横向き）
- 印刷専用ヘッダー（事務所情報・担当者連絡先）
- 印刷専用フッター（事務所住所）
- 操作ボタン・フィルター等は非表示（`.no-print` 系クラス）
- 提出済み書類の印刷非表示トグル
- カラーアクセントは印刷時に非表示

### 6.4 アクセシビリティ

- セマンティックHTML: `role`, `aria-expanded`, `aria-modal`, `aria-labelledby`
- キーボード操作: Enter/Space で展開、Escape でモーダル閉じ
- タブ順序管理

## 9. デプロイ

```bash
docker compose up -d
```

- http://localhost:3003/inheritance-tax-docs/ でアクセス
- ソースマウント + Vite HMR によるホットリロード

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build
```

- nginx:1.27-alpine による静的ファイル配信
- Nginx Gateway経由: http://localhost/inheritance-tax-docs/
